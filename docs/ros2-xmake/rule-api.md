# ament_xmake Rule API

The `ament_xmake.package` rule is the core integration between xmake and the ROS 2 ament build system. It handles package metadata installation, dependency resolution, CMake Config generation, and plugin registration.

## Rule Usage

Add the rule at the top of your `xmake.lua`:

```lua
add_rules("ament_xmake.package")
```

Both global mode (file scope) and target mode (inside a target block) are supported. Global mode is recommended — it applies to all targets and avoids duplicate installs.

## add_ros_deps(...)

Resolves ROS package dependencies and configures include directories, compile definitions, link flags, and rpath directories for the current target.

```lua
target("my_node")
    set_kind("binary")
    add_files("src/main.cpp")
    add_ros_deps("rclcpp", "geometry_msgs")
```

**Behavior:**
1. Reads package metadata from `_AMENT_XMAKE_ROS_INDEX` (generated by colcon-xmake at configure time)
2. Recursively resolves transitive dependencies declared in the index
3. Scans `AMENT_PREFIX_PATH` for include directories and libraries
4. Applies resolved includes, defines, link flags, and rpath to the current target

**Visibility option:**

By default, resolved dependencies are private to the target. To export them to downstream targets:

```lua
add_ros_deps("rclcpp", {visibility = "public"})
```

**Multiple dependencies:**

```lua
add_ros_deps("rclcpp", "std_msgs", "geometry_msgs", "sensor_msgs")
```

## install_ros_data(subdir, ...)

Registers additional data files for installation to `share/<pkg>/<subdir>/`.

```lua
install_ros_data("launch", "launch/*.py")
install_ros_data("config", "config/*.yaml")
```

**Parameters:**
- `subdir` — subdirectory under `share/<pkg>/` where files are installed
- `...` — one or more glob patterns matching files to install

**Auto-detection:** Even without explicit `install_ros_data()` calls, the rule automatically installs files from conventional ROS directories if they exist in the source tree:

| Directory | Description |
|-----------|-------------|
| `launch/` | Launch files |
| `config/` | Configuration files |
| `urdf/` | URDF robot descriptions |
| `meshes/` | 3D mesh files |
| `maps/` | Map files |
| `worlds/` | Simulation worlds |
| `rviz/` | RViz configurations |
| `params/` | Parameter files |

## install_ros_plugin(plugin_xml, base_class_pkg)

Registers a pluginlib plugin description XML file for installation and ament index registration.

```lua
install_ros_plugin("plugin_description.xml", "rclcpp_components")
```

**Parameters:**
- `plugin_xml` — path to the plugin description XML file (relative to source directory)
- `base_class_pkg` — the package that owns the base class (e.g., `rclcpp_components`)

**What it does:**
1. Installs the XML file to `share/<pkg>/`
2. Creates an ament index entry under `<base_class_pkg>__pluginlib__plugin/<pkg>` pointing to the installed XML

The rule also auto-detects XML files matching `*plugin*.xml` that contain `<library>` tags and installs them automatically.

## Install Rule Behavior

When a target with `ament_xmake.package` is installed, the rule performs these actions:

### Package Metadata (once per package)
1. Installs `package.xml` to `share/<pkg>/package.xml`
2. Creates ament index marker at `share/ament_index/resource_index/packages/<pkg>`
3. Copies headers from `include/` to the install prefix
4. Installs registered data files and plugins
5. Auto-installs conventional ROS data directories

### Per-Target Actions
- **Binary targets:** installed to `lib/<pkg>/<binary_name>`
- **Library targets:** installed to `lib/<library_file>` and a CMake Config is generated

### CMake Config Generation

For each library target, the rule generates:

1. **Per-target fragment** (`<target>-target.cmake`): defines an IMPORTED target `<pkg>::<target>`
2. **Top-level Config** (`<pkg>Config.cmake`): includes all fragments and provides legacy variables

```cmake
# Generated <pkg>Config.cmake provides:
set(<pkg>_FOUND TRUE)
set(<pkg>_INCLUDE_DIRS "...")
set(<pkg>_LIBRARIES "")
set(<pkg>_LIBRARY_DIRS "...")

# IMPORTED targets:
<pkg>::<target_name>   # Per-target (exact target name)
<pkg>::<pkg>           # Default alias (prefers shared, falls back to static)
```

Downstream CMake packages can consume these exports:

```cmake
find_package(demo_xmake_cpp REQUIRED)
target_link_libraries(my_node demo_xmake_cpp::demo_xmake_cpp)
```

## Symlink Install

Set the environment variable `AMENT_XMAKE_SYMLINK_INSTALL=1` to use symlinks instead of file copies during install. This speeds up iterative development on Linux.

## Rule Discovery

The rule is discovered through two mechanisms:

1. **Preferred (automatic):** colcon-xmake generates an internal entry file that includes the rule before loading your `xmake.lua`
2. **Fallback:** scans `AMENT_PREFIX_PATH` for `share/ament_xmake/xmake/rules/ament_xmake/package.lua`

In both cases, your `xmake.lua` simply calls `add_rules("ament_xmake.package")`.

## Deterministic Export

Only the target named exactly as the package name emits the default `<pkg>::<pkg>` alias in the CMake Config. Other targets are exported under their own names as `<pkg>::<target_name>`.
