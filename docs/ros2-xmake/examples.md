# Examples Walkthrough

The [ros2_xmake_examples](https://github.com/ros-x/ros2_xmake_examples) repository contains complete, buildable example packages demonstrating various xmake + ROS 2 patterns. This guide walks through each package.

## Setup

```bash
mkdir -p ~/ros2_xmake_ws/src && cd ~/ros2_xmake_ws/src
git clone https://github.com/ros-x/ros2_xmake_examples.git .
vcs import < ros2_xmake.repos

cd ~/ros2_xmake_ws
source /opt/ros/jazzy/setup.bash
colcon build --event-handlers console_direct+
source install/setup.bash
```

## demo_xmake_cpp — Multi-Target Package

A core example showing how a single `xmake.lua` can define multiple targets (static library, shared library, and executables) with ROS dependency resolution.

```lua
set_project("demo_xmake_cpp")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

target("demo_xmake_cpp")
    set_kind("static")
    add_files("src/demo_lib.cpp")
    add_includedirs("include", {public = true})
    add_headerfiles("include/(demo_xmake_cpp/*.hpp)")

target("demo_xmake_cpp_shared")
    set_kind("shared")
    add_files("src/demo_lib.cpp")
    add_includedirs("include", {public = true})
    add_headerfiles("include/(demo_xmake_cpp/*.hpp)")
    add_defines("DEMO_XMAKE_CPP_BUILDING_DLL", {public = false})
    set_basename("demo_xmake_cpp")

target("demo_xmake_cpp_node")
    set_kind("binary")
    set_installdir("lib/demo_xmake_cpp")
    add_files("src/main.cpp", "src/demo_lib.cpp")
    add_includedirs("include")

target("turtle_circle_node")
    set_kind("binary")
    set_installdir("lib/demo_xmake_cpp")
    add_files("src/turtle_circle_node.cpp")
    add_ros_deps("rclcpp", "geometry_msgs")
    add_includedirs("include")
```

**Key patterns:**
- `add_rules("ament_xmake.package")` enables the ROS integration rule
- `add_ros_deps("rclcpp", "geometry_msgs")` resolves ROS package includes and link flags
- `set_installdir("lib/demo_xmake_cpp")` places binaries in the standard ROS executable location
- Static and shared libraries are exported via CMake Config for downstream consumers

**Run the turtlesim demo:**
```bash
# Terminal 1
ros2 run turtlesim turtlesim_node
# Terminal 2
ros2 run demo_xmake_cpp turtle_circle_node
```

## demo_xmake_interfaces — rosidl Interface Package

Defines custom `.msg` and `.srv` files. The rosidl pipeline (handled by colcon-xmake) auto-generates C/C++ code — no explicit targets needed.

```lua
set_project("demo_xmake_interfaces")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

-- rosidl targets are auto-generated by the colcon-xmake rosidl pipeline
-- and injected via the entry file before this xmake.lua is loaded.
-- No explicit target definitions needed for interface-only packages.

-- Phony target to trigger the install rule
target("demo_xmake_interfaces_marker")
    set_kind("phony")
```

**Directory structure:**
```
demo_xmake_interfaces/
├── msg/
│   └── CustomMessage.msg
├── srv/
│   └── CustomService.srv
├── package.xml
└── xmake.lua
```

The `package.xml` must declare `<build_type>ament_xmake</build_type>` and include rosidl build dependencies. See the [rosidl doc](./rosidl.md) for details.

## demo_xmake_component — pluginlib Composable Node

Builds a shared library that registers as a composable node via `pluginlib`.

```lua
set_project("demo_xmake_component")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

-- Register the plugin description for pluginlib discovery
install_ros_plugin("plugin_description.xml", "rclcpp_components")

target("demo_xmake_component")
    set_kind("shared")
    add_files("src/talker_component.cpp")
    add_ros_deps("rclcpp", "rclcpp_components", "std_msgs")
```

**Key patterns:**
- `install_ros_plugin("plugin_description.xml", "rclcpp_components")` registers the plugin XML and creates the ament index entry for `rclcpp_components__pluginlib__plugin`
- The shared library is automatically installed to `lib/` and exported via CMake Config

## demo_xmake_mixed — C++/Python Mixed Package

A package containing both C++ nodes and Python modules.

```lua
set_project("demo_xmake_mixed")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

target("cpp_publisher")
    set_kind("binary")
    set_installdir("lib/demo_xmake_mixed")
    add_files("src/cpp_node.cpp")
    add_ros_deps("rclcpp", "std_msgs")
```

Python modules are installed via standard `setup.cfg` / `setup.py` mechanisms alongside the xmake-built C++ targets.

## demo_xmake_launch — Data Installation

A data-only package demonstrating launch file and configuration installation.

```lua
set_project("demo_xmake_launch")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

-- Install launch and config files to standard ROS locations
install_ros_data("launch", "launch/*.py")
install_ros_data("config", "config/*.yaml")

-- Dummy target so xmake has something to process
target("demo_xmake_launch_marker")
    set_kind("phony")
```

**Key patterns:**
- `install_ros_data("launch", "launch/*.py")` installs matching files to `share/<pkg>/launch/`
- Conventional directories (`launch`, `config`, `urdf`, `meshes`, etc.) are also auto-detected and installed even without explicit `install_ros_data()` calls
- A phony target is required so the install rule fires

## demo_xmake_custom_msg_node — Cross-Package Message Consumption

A node that publishes custom messages defined in `demo_xmake_interfaces`, demonstrating cross-package interface consumption.

```lua
set_project("demo_xmake_custom_msg_node")
set_version("0.1.0")
set_languages("cxx17")

add_rules("mode.debug", "mode.release")
add_rules("ament_xmake.package")

target("custom_msg_publisher")
    set_kind("binary")
    set_installdir("lib/demo_xmake_custom_msg_node")
    add_files("src/publisher_node.cpp")
    add_ros_deps("rclcpp", "demo_xmake_interfaces")
```

Simply add the interface package to `add_ros_deps()` — the generated headers and typesupport libraries are resolved automatically.

## downstream_cmake_consumer — CMake Interoperability

A standard `ament_cmake` package that consumes the library exported by `demo_xmake_cpp`, proving that xmake-built packages produce valid CMake Config files.

```cmake
find_package(demo_xmake_cpp REQUIRED)
target_link_libraries(my_node demo_xmake_cpp::demo_xmake_cpp)
```

This works because `ament_xmake.package` generates `<pkg>Config.cmake` with IMPORTED targets during install.

## Build Order

colcon resolves the dependency graph automatically:

```
ament_xmake (build tool)
  └─ demo_xmake_cpp
  │    └─ downstream_cmake_consumer
  └─ demo_xmake_interfaces
  │    └─ demo_xmake_custom_msg_node
  └─ demo_xmake_component
  └─ demo_xmake_mixed
  └─ demo_xmake_launch
```

Build all packages:
```bash
colcon build --event-handlers console_direct+
```

Build a specific package:
```bash
colcon build --packages-select demo_xmake_cpp --event-handlers console_direct+
```

Run tests:
```bash
colcon test --packages-select demo_xmake_cpp downstream_cmake_consumer
colcon test-result --verbose
```
